## Source-vondst

In `SecondReality_source/GLENZ/MAINTRAN.C` (de "translucent" variant — naast `MAINNORM.C` voor opaque):
- **Mesh** = kubus 2000×2000×2000 (`points[]` regel 11-19), 8 verts ±1000
- **12 edges** + **6 quad-faces** met kleur-codes `0x4004/8/10/20/40/80`
- **Pipeline:** `cmatrix_yxz → csetmatrix(0,0,4000) → crotlist → cprojlist → cpolylist → cdrawpolylist`
- **Rotatie:** `rz+=5; ry+=7; rz+=6` per frame in tenths-of-degree (= `ry+=7, rz+=11` netto, rx blijft 0)
- **Translucentie-truc:** geen echte alpha-blend, maar **palet-bit-OR**: elke face zet één bit (0x04 R, 0x08 G, 0x10 B, 0x20 RG, 0x40 GB, 0x80 BR), palet-init zet RGB-componenten per bit zodat overlappende pixels door bit-OR de som van kleuren tonen (`MAINTRAN.C:60-69`). Geïmplementeerd via mode-X tweaked polygon-fill in `VIDTWE.ASM`.

## Modern reinterpretatie keuzes

- **Echte additive blending** i.p.v. palet-OR (visueel identiek effect, simpeler code)
- **Resolutie-onafhankelijk** (geen Mode-X 320×240 non-square pixels)
- **Square pixels** + 60° FOV (1993 origineel onbekend exacte FOV)
- **Alpha 0.55** als compromis: hoger = te wit overlap, lager = te dof solo (kan getuned later)
- **Camera op z=-4** (mesh ±1, dus relatieve afstand 4 = ~origineel `csetmatrix(0,0,4000)` met mesh ±1000)
- **Geen edges** (alleen filled faces) — wireframe overlay parkeren voor 0.2.x
