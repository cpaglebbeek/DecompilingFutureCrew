## Waarom chiptune3 in `public/` i.p.v. npm-dep

Eerste poging: `npm install chiptune3` + ES import. Vite bundelde de wrapper, maar de chiptune3.worklet.js doet **inside de worklet** een `import libopenmptPromise from './libopenmpt.worklet.js'`. Vite analyseert AudioWorklet-modules niet recursief voor imports, dus `libopenmpt.worklet.js` belandde niet in `dist/`. Bij runtime crash: 404 op libopenmpt.worklet.

Oplossing: 3 chiptune3 files in `public/chiptune3/` plaatsen waar Vite ze 1-op-1 kopieert. De relative `import './libopenmpt.worklet.js'` werkt dan native in de browser (AudioWorklet ES module support sinds 2023). Audio-engine doet dynamic `import()` naar `${BASE_URL}chiptune3/chiptune3.js`. npm-dep verwijderd voor zuivere repo-staat.
